1. Visão Geral do Produto

CNPJTrack é uma plataforma SaaS web voltada exclusivamente para contadores e escritórios de contabilidade brasileiros. O sistema consulta dados fiscais de empresas diretamente na Receita Federal (via BrasilAPI com fallback para ReceitaWS), armazena o histórico de alterações e alerta o contador quando qualquer mudança é detectada — regime tributário, status cadastral, entrada/saída do Simples Nacional.

Problema central: Contadores precisam verificar manualmente a situação fiscal de cada cliente todo mês para enviar obrigações à Receita. Esse processo é feito um a um, sem histórico, sem alertas e sem rastreabilidade. O CNPJTrack elimina esse trabalho repetitivo.

3.4 Cache de 24 horas

Quando um CNPJ é consultado, o sistema verifica se já existe registro com last_checked_at nas últimas 24 horas. Se existir, retorna o dado do banco sem chamar APIs externas (status: "cached"). O parâmetro force: true ignora o cache — usado na re-verificação manual do Monitoramento.

3.5 Detecção de Mudanças

Campos rastreados comparados a cada consulta:

razao_social simples_nacional regime_tributario status_cadastral data_inicio_regime

Quando há diferença entre o valor anterior e o novo, o sistema grava em company_changes com valor antigo, novo e timestamp.

5. Edge Functions (Backend Serverless)

consulta-cnpj

Principal função do sistema. Recebe array de até 100 CNPJs e processa cada um:

  1. Verifica autenticação (token Bearer obrigatório)
  2. Verifica cache de 24h no banco para cada CNPJ
  3. Se sem cache: chama BrasilAPI (timeout 8s, 2 tentativas)
  4. Se BrasilAPI falhar: chama ReceitaWS como fallback (2 tentativas)
  5. Detecta mudanças comparando 5 campos com o registro anterior
  6. Salva/atualiza no banco com user_id
  7. Grava mudanças em company_changes
  8. Registra usage_logs com quantidade de consultas bem-sucedidas
  9. Retorna results[] + summary

Normalização: Status mapeia textos da Receita para 5 valores padronizados. Regime deriva de porte + flag MEI + flag Simples.

mp-create-preference

Cria preferência de pagamento no Mercado Pago. Valida autenticação, valida plano (Starter ou Pro), busca plan_id no banco, cria preferência com item/payer/URLs de retorno/webhook URL, salva subscription pending, retorna init_point.

mp-webhook

Recebe notificações do Mercado Pago. GET retorna 200 (verificação). POST: busca dados do pagamento na API do MP; se approved, extrai userId|plan_name do external_reference e ativa o plano por 30 dias via upsert em user_plans.

Segurança das Edge Functions: CORS por allowlist de origens, Bearer token obrigatório, verificação de usuário no Supabase antes de processar qualquer dado, origens não autorizadas recebem HTTP 403.
